THEORY & IMPLEMENTATION

Architecture Deep-Dive

Click on any component below to reveal the deep technical explanation of how SecureFedHE achieves decentralized privacy.

1

Selective Homomorphic Encryption (TenSEAL CKKS)

+

The Problem: Applying Fully Homomorphic Encryption (FHE) to an entire Deep Neural Network causes a 1000x computational slowdown, making it impossible to run on edge devices like hospital computers.

The Solution: We mathematically isolated the neural network layers. We only apply heavy TenSEAL CKKS encryption to the final classification layer (FC2). By leaving the massive convolutional blocks in plaintext, we bypass the FHE bottleneck entirely while maintaining cryptographic security on the final prediction.

2

Differential Privacy (Gaussian Mechanism)

+

The Problem: If we don't encrypt the early layers of the Neural Network, attackers could use "Gradient Inversion Attacks" to reconstruct the original patient data from the plaintext weights.

The Solution: We inject mathematical Gaussian Noise into the intermediate layer (FC1). This is called Differential Privacy. The noise is carefully calibrated using an Epsilon (ε) budget. It obscures the exact data points enough to prevent reconstruction attacks, but preserves enough statistical patterns for the AI to learn. We tested ε=10, 20, and 50 to find the perfect privacy-utility trade-off.

3

True Distributed Ring Topology (Peer-to-Peer)

+

The Problem: Standard Federated Learning relies on a Central Aggregation Server (a "Star Topology"). If that central server is hacked or crashes, the entire global AI network collapses.

The Solution: We deleted the central server. Using Flask web servers, we built a decentralized Ring Topology. Hospital A encrypts its weights into a Base64 string and sends them directly to Hospital B over HTTP. Hospital B mathematically adds its local encrypted weights to the payload without decrypting it, and passes it to Hospital C. Once the payload returns to Hospital A, the global model is fully aggregated with zero central authority.
