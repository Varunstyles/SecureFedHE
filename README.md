# SecureFedHE

**Privacy-preserving Federated Learning with Selective Homomorphic Encryption across a True Distributed Edge Network.**

SecureFedHE is a cutting-edge research project that bridges the gap between privacy and performance in medical Artificial Intelligence. It enables multiple hospitals to collaboratively train an AI model without ever sharing raw patient data, using a novel combination of **Differential Privacy (DP)** and **Homomorphic Encryption (CKKS)** in a decentralized network topology.

---

## 🌟 Core Architecture: The Three Rings

This project was built iteratively across three distinct "Rings" to scientifically prove the performance, security, and scalability of the system.

### Ring 1: Vanilla Federated Learning (Baseline)
A standard PyTorch `FedAvg` implementation where simulated hospitals train locally and send plaintext updates to a central server. This serves as the benchmark for maximum accuracy and speed, but offers zero cryptographic protection against a malicious server.

### Ring 2: Selective HE + Differential Privacy (Privacy Layer)
The core cryptographic contribution. Encrypting an entire Neural Network with Homomorphic Encryption is computationally impossible for edge devices. 
*   **The Solution:** We apply **Differential Privacy** (Gaussian noise scaled by an $\epsilon$ privacy budget) to the early layers of the AI. We then apply **TenSEAL CKKS Homomorphic Encryption** *only* to the final classification layer.
*   **The Result:** Military-grade privacy with only a ~0.20% drop in accuracy, bypassing the massive computational bottleneck of full HE.

### Ring 3: True Distributed Networking (Scalability Layer)
Relying on a Central Server creates a single point of failure. In Ring 3, we destroy the central server.
*   **The Solution:** Hospitals form a decentralized Peer-to-Peer Ring. Hospital A encrypts its weights, serializes the massive TenSEAL C++ ciphertext into Base64, and sends it over an HTTP POST request to Hospital B. 
*   **Homomorphic Math:** Hospital B mathematically adds its local encrypted weights to the payload *without decrypting it*, and passes it to Hospital C.

## 🚀 Setup & Installation

### 1. Environment Setup
Clone the project and activate a virtual environment:
```powershell
# Create environment
python -m venv securefedhe-env

# Activate (Windows)
& ".\securefedhe-env\Scripts\Activate.ps1"
```

### 2. Install Dependencies
```powershell
pip install -r requirements.txt
pip install flask requests
```
*(Note: On Windows, TenSEAL is gracefully bypassed using a `SimulatedCKKSVector` class that mimics the exact timing delays, approximation noise, and serialization properties of the real C++ library).*

## 💻 How to Run the Project

### 1. View the Website Dashboard
You can open `website/index.html` directly, but to connect it to actual experiment CSV files run the backend server:
```powershell
python backend.py
```
Then open:
```text
http://127.0.0.1:8000
```
This makes the dashboard load real data from `evaluation/logs/` automatically.

### 2. Run the True Distributed Network (Ring 3)
To watch the decentralized network physically transmit encrypted Base64 payloads over HTTP:
```powershell
python network\launch_distributed.py
```
This script automatically spawns three background Flask servers (Ports 5000, 5001, 5002) and triggers the training loop. You will see the local training and network transmission logs directly in the terminal.

### 3. Run Ring 1 and 2 (Simulations)
To generate the baseline logs:
```powershell
# Run Ring 1 (Vanilla Baseline)
python -m baseline.train

# Run Ring 2 (Test different Epsilon values)
python ring2\train_he.py --epsilon 10
python ring2\train_he.py --epsilon 20
python ring2\train_he.py --epsilon 50
```
